AlignUsingReferencePoints¶

Performs Reference Point System (RPS) based alignment.

class AlignUsingReferencePoints¶

Bases: geomagic.api.v3.swig.ConstrainedAlign

Attributes

Name

Read only

Type

Range

Default Value

Usage hints

Description

featurePairs

No

DataObjectArray

Input (required)

List of AlignmentFeaturePair data objects. On the fixed side these features must be pairable, that is, each feature must uniquely identify a reference point. These feature types are: Point Target Features, Point Features, Line Target Features (identifying the contact point), Circles, Slots, Rectangles, and Rounded Rectangle Features (because they have center points).

constraint

No

AlignmentConstraint

None

Input (optional)

It’s an optional Alignment Constraint. If this property is not given or every degree of freedom is free then an unconstrained alignment is performed.

doFeatureAlign

No

bool

True

Input (optional)

If this property is True then initially a feature based alignment (see AlignUsingFeatures) will be performed in order to get a starting alignment transformation from where the actual iteration will be able to proceed. (The user can get obtain the result transformation of this initial feature alignment from featureAlignTransform.) If this property is False then it is assumed that the input fixed features are already in the proper initial position.

fixedCenter

No

Vector3D

[0.00 0.00 0.00]

Input (optional)

Origin of the fixed model.

floatingCenter

No

Vector3D

[0.00 0.00 0.00]

Input (optional)

Origin of the floating model.

floatingObject

No

DataObject

None

Input (optional)

It must be specified only if there is at least one target (TargetPoint or TargetLine) on the featurePairs list. If it is given, it should be a Points, GriddedPoints, or Mesh object that is to be aligned to the fixed features. Note that here a Points object must have oriented normals.

maxIter

No

int

0 - 10000

120

Input (optional)

Maximum number of alignment iterations.

int

20

Minimum number of alignment iterations.

int

25

Input (optional)

Number of target reevaluations.

tolerancedIterations

No

int

0 - 100

0

Input (optional)

Number of toleranced iterations (for weighted alignment).

testTransform

No

Transform3D

identity

Input and Output

The result of the alignment. On input it contains the initial transformation of the floating object to the fixed. On output it contains the full alignment transformation.

alignmentStatus

Yes

enumeration

AlignmentConverged

Output

Contains the outcome of the algorithm: AlignmentConverged or AlignmentDiverged or TargetReevaluationFailed or AlignmentFailed.

angularDev

Yes

DoubleArray

Output

An array containing the angular deviations between each feature pair.

distanceDev

Yes

DoubleArray

Output

An array containing the distance deviations between each feature pair.

featureAlignTransform

Yes

Transform3D

[1.00 0.00 0.00 0.00] [0.00 1.00 0.00 0.00] [0.00 0.00 1.00 0.00] [0.00 0.00 0.00 1.00]

Output

This transformation is the result of the optional feature based alignment at the beginning of the algorithm (see doFeatureAlign). The final result in testTransform will be the composition of this and the subsequent reference point based alignment. If doFeatureAlign is False then this will be a unit transformation.

updatedFeatures

Yes

DataObjectArray

Output

List of the reevaluated Features data objects on the original model. (They are at the updated position in the original coordnate system.)

Description

Algorithm diverged.

AlignmentFailed

The alignment failed.

TargetReevaluationFailed

At some point of the algorithm the system failed to reevaluate (update) the target. It did not meet the model.
